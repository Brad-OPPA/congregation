## B. 모범 패턴 한 페이지 요약

### 1. 글자수
- **권장 범위**: 2,090 ~ 3,600자
- 원준님 직접 작성(3-7번): 평균 2,486자 (콤팩트)
- 모범 빌드 검수 통과(1-2번): 평균 3,344자 (충실)
- **하한 2,000자 / 상한 3,600자** — 실패 사례 4,100자 = 명백히 초과

### 2. 본문/예/적용 비율
- **권장**: 본문 50-70 / 예 8-25 / 적용 19-40
- 11개 평균 (성공 1-10): **52 / 17 / 31**
- 실패: 45 / 38 / 17 → 예가 두 배 / 적용 반토막

### 3. 시간 마커 (의무 ⭐)
- **모든 자료에 박힘** — 평균 3.8개 (성공권 3.9, 실패 3)
- 가장 흔한 패턴: `1'30"` (서론 끝) + `8'30"` 또는 `9'30"` (결론) + 중간 1개 = 3개
- 풍부 패턴: 서론 + 요점 1·2·3 진입 시점 + 예 시작 + 결론 = 5~6개
- **권장**: 최소 3개 (서론·중간·결론) / 권장 5~6개 (요점별·예 진입 분리)
- **위치 의무**: 서론 끝 + 결론 직전 + 중간 (요점 전환·예 도입) 최소 1개
- 형식: `N'M"` (유니코드 prime `′″` 또는 ASCII `'"` 모두 OK), 빨강·볼드

### 4. 출처 호명 (= "「파24.07」 30면이 짚는 것처럼" 류)
- **원준님 직접 표준 (3-7) = 정확히 0건** ⭐
- 모범 빌드 (1·2) = 7~8건 (검수 통과지만 정확성 점검 권장)
- **하한 0 / 상한 7** — 실패 17건 = 8배 초과
- **결론**: 0이 정확. 6 이하면 허용. 7 이상 의심.

### 5. 외부 14축 (키루스·요세푸스·고고학·과학·통계 등)
- **모든 모범 자료 = 정확히 0건** ⭐⭐ — 절대 기준
- 실패 = 18건 본문 침입
- **결론**: 본문에 외부 14축 등장 = **즉시 NG**

### 6. 청중 적용 단락 수
- 권장 범위: 3 ~ 11 단락 (메시지에 따라 탄력)
- 원준님 표준(3-7): 평균 4.0
- 모범 빌드(1-2): 평균 10.0
- **하한 3** — 실패 4건이지만 비율 17%로 너무 적음

### 7. 핵심 한 문장 (서론·결론 박힘)
- **모든 모범 자료 = 0** — 동일 한 문장 박힘 X
- **계획 §3.2 가설 수정 필요**: "핵심 한 문장이 서론·결론에 박힘" 은 표준 X. 대신 **톤과 방향이 일관됨**.

### 8. 타종교 키워드 (불교·이슬람·힌두·천주교·개신교 등)
- **모든 모범 자료 = 0** ⭐⭐
- **결론**: 0이 의무. 1건이라도 NG.

### 9. 6단계 narrative (도입→성구→낭독→설명→예→적용)
- 원준님 표준(3-7): 평균 5.4 / 6
- 모범 빌드(1-2): 평균 5.0 / 6
- 참고(8-10): 평균 4.0 / 6
- **권장**: 5/6 이상 / **하한 4.5/6**

### 10. 이미지

## G. 본질 구조 — 6단계 narrative + 결론 정책 (2026-05-01 원준님 직접 확정)

### G-1. 출판물의 역할 = 성구의 배경 설명

각 요점 = [성구] + [그 성구를 풀어주는 출판물 (「파」·「통」·「집교」) 배경 설명].
출판물 자체가 본문이 아니라 **성구 안 단어·문구·문맥의 배경 설명** 자료.

### G-2. 각 요점의 6단계 narrative (의무)

| 단계 | 내용 |
|---|---|
| 1. 흥미 유발 예 | 주제에 적절한 예 (출판물에 있으면 좋고, 없어도 다양한 예 OK) |
| 2. 질문 | 그 예에서 요점으로 환기 |
| 3. 답 = 성구 | 질문의 답이 성구에 나옴 |
| 4. 성구 낭독 | verbatim |
| 5. 단어·문구 풀이 | 성구 안 단어·문구를 「파」·「통」·WOL 검색해서 의미 풀어냄 |
| 6. 다음 요점 연결 | 다음 요점으로 자연스럽게 |

요점 안 보조: 간단한 예·비유로 이해 돕기 OK.

### G-3. 결론 단계 (3가지 의무)

1. **집회 교재 (집교) 본 주차 삽화 활용**
   - 집교 4면 삽화 보여주기
   - 그 삽화에 배운 3 요점이 어떻게 녹아 있는지
   - 삽화 배경 + 삽화를 통한 배울 점

2. **전체 내용 간단 요약** — 주제와 연관해서 짧게

3. **서론 연계 (콜백)** — 서론에서 던진 예·질문·이미지를 결론에서 다시 환기해서 닫음

### G-4. 책무 분리 — 기획자 = 기획만 / script = 작성만

- **planner** ① 단계 (지시서) + ③ ⑤ 단계 (검수만)
- **script** ④ 단계 (본문 작성)
- planner 가 본문 작성하면 정책 위반 (memory `feedback_planner_no_writing.md`)

### G-5. 보조 리서치 = 광범위·풍부 검색 의무

자료 부실 = 본문 부실 = 외부 14축·추측 보충 (실패 패턴).

베이스라인 (직전 모범 자료 기준):
- scripture-deep: 성구 ≥ 6개 깊이 자료
- publication-cross-ref: 출판물 ≥ 5편 인용
- illustration-finder: 후보 ≥ 6개 (요점당 2개) + 결론 집교 삽화 1장
- experience-collector: 경험담 ≥ 5개
- application-builder: 적용 카드 ≥ 4개

부족 시 planner 가 ③ 단계에서 보조 재호출 (작성으로 보충 X).
