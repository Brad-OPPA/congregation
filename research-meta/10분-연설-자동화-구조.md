```
   ↓

   ↓
[④ treasures-talk-script]
   - 본 주차 「파」 단락 골격 추출
   - 서론 5 흐름 작성 (청중 질문 → 흥미 → 주제 → 환기 → 안내)
   - 요점 1·2·3 = 시간 마커 직후 성구 회색 박스 → 짧은 환기 → 단어 풀이 → 적용 → 환기·다음 안내
   - 결론 5 단락 (요점 리마인드 + 자문점 + 서론 콜백 + 적용 호소 + 마무리 약속)
   - 모범 정형 표현 박힘 ("함께 OO장 OO절을 확인해 보겠습니다" 등)
   ↓
[⑤ assembly-coordinator] — 옵션 B
   - script.md → spec dict 변환
   - R1~R18 자체 grep (1차)
   - content_{YYMMDD}.py 드래프트
   ↓
[⑥ planner 2차 재검수] — R1~R18 검증
   ↓
[⑦ 빌드] — python3 content_{YYMMDD}.py
   - build_treasures_talk.py (Path.home() OS 자동 감지)
   - LibreOffice soffice → docx + PDF
   - 출력: ~/Dropbox/02.WatchTower/01.▣ 수원 연무 회중/01.주중집회/01.성경에 담긴 보물/01.10분 연설/{YYMMDD-MMDD}/
   ↓
[⑧ 🚨 4종 게이트 자동 호출] — 메인 책무, 한 메시지에 병렬
   - fact-checker (사실·인용·성구 표기) → research-factcheck/
   - jw-style-checker (WOL 최근 10년 + 사용자 NG list + 의심 어휘 grep) → research-style/
   - timing-auditor (낭독 시간 ±120초) → research-timing/
   - quality-monotonic-checker (직전 주차 단조 증가 9축) → research-quality/
   ↓
[⑨ 결과 처리]
   - 4종 모두 PASS → 사용자 검수 단계로
   - 1건 이상 FAIL → 자동 재작성 (해당 영역 재호출, 5회 한도)
   - 5회 초과 → 사용자 BLOCKING 알림
   ↓
[⑩ 사용자 검수 1회] — docx/PDF 직접 확인
   ↓
[완료] 메일 발송 또는 다음 작업
```
