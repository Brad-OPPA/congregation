## 4. publication symbol 분리 정책 (260514 빌드 발견, 정본 확정 2026-05-02)

회중 통칭과 실제 publication symbol 이 다름:

| 회중 통칭 | docid 접두사 | 실제 symbol | 실제 책 제목 (영문) |
|---|---|---|---|
| 「훈」 | `1102016XXX` | **`lfb`** | Learn From the Bible / 「내가 좋아하는 성경 이야기」 (어린이용 그림 이야기책, 2017) |
| 「예수」 책 | `1102014XXX` | `jy` | Jesus — The Way, the Truth, the Life / 「예수 — 길, 진리, 생명」 |

**핵심 사실**:
1. 5/14 주차 「훈」 84-85장 = lfb 84-85장. 「예수」 책 (jy) 의 동일 일화 (44/53/71장) 와 별도 책.
2. lfb 본문은 어린이용 표현 — 사회자 보강 멘트로 깊이 추가 (성경 배경·고고학·원어 활용).
3. **fact-checker 는 publication symbol 검증 의무**. 옛 docx (260205·260423·260507) 의 jy 표기는 잘못. 신규 빌드부터 lfb 정본.
4. **script 안 표기 분리**:
   - 전면 (교재 본문) = `「훈」 NN-MM장` 또는 `「내가 좋아하는 성경 이야기」 NN-MM장`
   - 횡단 인용 (성인용 깊이) = `「예수」 책 44장`, `「예수」 책 53장` 등 — **반드시 "예수 책" 명시**

5. cbs-planner.md (에이전트 명세) 의 "「모든 좋은 일에 전념할 수 있도록 훈련된 사람」" 표기는 **별도 책 (다른 사이클의 교재)** 으로, 현재 사이클 (2026) 과 무관. 사이클 전환 시 wol 인덱스가 새 교재 자동 지시.

## 9. 4종 게이트 종합 판정 우선순위

```
1. fact-checker HIGH 1건 이상 → NEEDS-FIX (재빌드 강제)
2. jw-style-checker HIGH 1건 이상 → NEEDS-FIX (재빌드 강제)
3. quality-monotonic-checker FAIL → 자동 재작성 (5회 한도)
4. timing-auditor FAIL → quality PASS 면 통과 (quality > timing 정책)
```

5건 모두 HIGH 0 / MED ≤ 2 / LOW 무관 → 사용자 검수 단계 진입.
