## 🎯 핵심 원칙 (2026-05-09 v3)

**현재 단계 = 퀄리티 올리는 중** — 사용자 마음에 쏙 들 때까지.

흐름:
1. **현재 (마음에 쏙 들기 전)** — 매 빌드 직전 대비 향상. 단조 증가 의무. 표준 §1~§5 모두 충족 노력.
2. **사용자가 "쏙 든다" / "OK" 시점** — 그 빌드를 OK baseline 으로 등록.
3. **OK 시점 이후** — 그 OK baseline ≥ **85%** 유지. 무리한 추가 상승 X (시간 낭비).

OK baseline 등록 방법:
- 사용자가 명시적으로 "OK" / "쏙 든다" / "이 정도면 됐다" 라고 한 docx
- 메인 Claude 가 즉시 표시 (해당 docx 를 §5.1 좋은 사례로 추가, 메트릭 baseline 으로 동결)

규칙:

- 본문 누락은 무관용 (§2.4) — OK 시점 무관
- 매번 새 Claude 와 같은 이야기 반복 X — SessionStart 자동 인식 의무

## 1. 형식 표준 (빌더 코드에 박힘)

### 1.1 레이아웃
- **줄높이**: 1.0 (예: build_watchtower.py `line_spacing=1.0`)
- **불필요한 줄바꿈 금지**: 단락 사이 `space_after=1~2pt` 만 (3pt 이상 금지)
- **본문 폰트 크기**: 11~14pt (슬롯별 디폴트 따름)

### 1.2 시간 마커
- **색상**: 빨강 글자 + **노랑 배경** (highlight=yellow)
- **위치**: 우측 정렬, 볼드
- **단위**: 분 단위 (예: `2'`, `5'`, `8'`)

### 1.3 마크업 표준
- **중요 키워드**: 볼드 처리 (단락 안 핵심 단어 1~3개)
- **중요 구**: 노랑 하이라이트 (강조하고 싶은 짧은 구절)
- **성구 verbatim**: 빨강 (#FF0000)
- **출판물 인용 「」**: 볼드
- **사회자 대사**: 노랑 하이라이트
- **소제목**: 진한 볼드 + 네이비 색상 (#1F3864) 가운데 정렬

### 1.4 삽화
- **WOL 본문 삽화 모두 임베드** (생략 금지)
- 각 삽화 = 이미지 + 캡션 + 배경 설명 + 해설 4단 구조
- **삽화 해설은 본문과 연결되어야 함** (단순 캡션 X)

## 2. 깊이 표준 (콘텐츠 의무)

### 2.1 핵심 성구 해설
**한 항 = 인용된 모든 성구 깊이 다룸** (1개만 X).

각 성구별로:
- ✅ 신세계역 연구용 성서 (nwtsty) verbatim
- ✅ **연구 노트 (study note) 인용** — wol.jw.org 연구용 성서 페이지에서
- ✅ **상호 참조 성구 (cross-references)** — 의미 보강
- ✅ **항 본문과의 연결 해설** — "이 성구가 왜 이 항에 인용됐는가" 명시
- ✅ 출판물 (「파수대」·「통찰」·「예수」 등) 의 해당 성구 해석 인용

**금지**:
- ❌ 성구를 단순 verbatim 만 박고 해설 없음
- ❌ 항당 1개 성구만 깊이 — 항에 등장하는 모든 성구 다뤄야

### 2.2 출판물 인용 (가장 적합한 1개 선정 원칙)
**광범위 탐색 → 적합성 1개 선정**:

각 항·각 성구에 대해:
1. **광범위 탐색** — 다음 출판물 모두 wol 횡단 검색:
   - 「파수대」 (연구용 + 공중용) 과거호 (전 연도)
   - 「통찰」 1·2권 해당 표제어
   - 「예수 — 길, 진리, 생명」
   - 「하느님의 사랑 안에 머무십시오」
   - 「여호와께 가까이 가십시오」
   - 「깨어라!」 관련 기사
   - JW 방송 자료
2. **적합성 평가** — 후보 중 다음 기준으로 1개 선정:
   - 항 주제와 직접 연결
   - 성구 의미 보강
   - 청중 (회중) 적용성
   - 최근성 (가능하면 최근 10년)
3. **인용 시 명시**:
   - 출판물 이름 + 호수/면 (예: "「파수대」 2024년 1월호 8면 14항")
   - 인용 verbatim
   - **"왜 적합한지" 짧은 사회자 멘트**

**최소 메트릭**:
- 항당 적합한 출판물 인용 ≥ 1 (심도 보장)
- 전체 docx 출판물 인용 ≥ 항 수 × 1.0 (예: 17 블록 → 17+ 출판물 인용)

### 2.3 참조 성구 비교 (NEW — 사용자 명시 2026-05-09)
**한 항 안의 모든 성구를 살펴야 함**:

각 항에 등장하는 모든 성구 (본 성구 + 인용 성구 + 참조) 에 대해:
- ✅ 모두 신세계역 연구용 verbatim
- ✅ 항 본문이 그 성구의 어떤 측면을 강조하는지 해설
- ✅ 성구들이 서로 어떻게 연결되는지 (보완·대조·점층) 분석

**금지**:
- ❌ 항에 5개 성구 있는데 1개만 깊이 다루고 나머지 무시
- ❌ "이 항의 핵심 성구는 X" 라고 하나만 픽업

### 2.4 본문 누락 금지 (모든 슬롯 공통, 2026-05-09 추가)

**WOL 본문에 있는 요소는 모두 docx 에 포함 의무** — 빠지면 안 됨.

각 슬롯별 본문 = WOL 페이지의 실제 내용:
- 성구 (본 성구·인용 성구·참조 성구) — 모두 verbatim 포함
- **삽화** — figure / illustration / .caption 모두 임베드 + 캡션 + 해설
- 출판물 인용 — 본문에 명시된 「」 인용 모두
- 질문 — 본문 질문 / 복습 질문 모두 (파수대의 경우 복습 질문 그대로)
- 소제목 / 항 번호 — 모두 표시
- 시간 마커 — 본문 구조에 맞게

**금지**:
- ❌ "이건 작은 거니 생략" — 본문 1줄도 무단 생략 금지
- ❌ 자동 빌드가 누락한 요소 그대로 두기 — spec 보강해서 채워야

**일률 적용 X**:
- 모든 슬롯이 같은 형식 강제 X
- 각 슬롯의 본문 구조에 따라 적합하게 (예: 파수대 = 17 블록 + 4 소제목 + 3 복습 / CBS = 30분 사회자 + 시간마커 8개 / 10분 연설 = 4페이지 단편)
- **공통 원칙**: 그 슬롯 본문에 있는 것은 모두 포함

### 2.5 적용점 깊이 (4축 의무)
**모든 항의 적용 단락이 다음 4축 중 최소 2~3 축 등장**:

1. **회중** — 집회·서로 격려·일치
2. **전도 (야외 봉사)** — 구역·재방문·연구
3. **가정** — 가족 성서 연구·자녀 양육·부부
4. **개인 영성** — 기도·연구·묵상

**의무**:
- ✅ 각 적용 단락은 출판물 인용 기반 (창작 X)
- ✅ 자기점검 질문 1개 포함 권장
- ✅ 추상적 권고 X — 구체적 실천 1~2 단계

**최소 메트릭**:
- 적용 단락 수 ≥ 항 수의 50% (예: 17 블록 → 8+ 적용 단락)
- 4축 모두 docx 안에 최소 1번 등장

## 3. 자동 검사 메트릭 (빌드 직후 강제)

`quality_monotonic_check.py` 가 다음 모두 검증. 미달 시 NO-GO + 재작성 권고.

### 3.1 정량 메트릭 (절대 하한)
| 메트릭 | 절대 하한 (슬롯별 다름) |
|---|---|
| 글자수 | 직전 평균 × 0.95 |
| 「출판물」 인용 | **항 수 × 1.0** (예: 17 블록 → 17+) |
| 성구 인용 | 직전 평균 × 0.95 |
| 이미지 | 직전 평균 (생략 금지) |
| 깊이 단락 (100자+) | 직전 평균 × 0.95 |

### 3.2 패턴 메트릭 (정성)
- 시간 마커 노랑 배경 ≥ 1 (없으면 빌더 깨짐)
- 적용 4축 키워드 (회중·전도·가정·개인) docx 안 등장 ≥ 3축
- 각 성구 인용 후 해설 단락 (50자+) 존재

## 4. 사용자 학습 누적 (자산 쌓이는 방식)

### 4.1 어휘 학습 — 자동
사용자가 "이 어휘 또 쓰지마" 한 번 → `add_user_ng.py` → `banned-vocabulary.md` 영구 추가
→ 다음 빌드부터 agent-postcheck-hook 가 자동 차단.

### 4.2 형식 표준 학습 — 즉시 적용
사용자가 "줄높이/시간 마커/색상" 지적 → 빌더 코드 즉시 패치 + 본 파일 갱신.

### 4.3 깊이 표준 학습 — 표준 정본 추가
사용자가 "이런 깊이가 부족" 지적 → 본 파일 §2 (깊이 표준) 에 항목 추가
→ agent-prebrief-hook 가 다음 빌드 prompt 에 자동 prepend.

### 4.4 사례 학습 — 좋은/나쁜 예시 누적
좋은 사례·나쁜 사례를 본 파일 §5 에 누적 → 에이전트가 학습.

### 6.1 파수대 (week-study) — 오프닝/결론 구조 (2026-05-09 갱신 v2)

**오프닝 (서론) — 약 1분**:
- 깊은 해설 X (오프닝은 본문이 아님)

- 분량 기준 (문장 수 기준 X — 자연스럽게 1분 분량)

**금지**:
- ❌ 오프닝에서 출판물 인용 깊이 보강 (X — 본문에서 충분)
- ❌ 5~10분 길게 (X)

**결론 — 약 1분**:
- 오늘 배운 내용에서 어떤 중요한 것을 배웠는지 환기
- 적용 포인트 짧게
- 마침 노래 + 기도 안내
- **분량 기준 약 1분** (문장 수 의무 X)

**금지**:
- ❌ 결론에서 새 출판물 인용 (X — 본문에 다 있음)
