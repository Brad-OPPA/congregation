## 4. 산출물 존재 시 skip 정책

스킬 진입 시 출력 폴더에 산출물이 있는지 먼저 확인.

- **없음**: 정상 진행.
- **있음 + 사용자 무명시**: 단정형 확인 1회 ("이미 있는데 새로 만드시나요?") → 답 없거나 No → **skip**.
- **있음 + 사용자가 "재생성·업그레이드·버전 올려·다시 만들어" 명시**: 파일명 끝에 `_verN_` 부여 후 신규 생성 (기존 파일 보존).
  - **N 자동 결정**: 같은 폴더에서 같은 prefix 의 파일 중 가장 큰 `_verN_` 을 찾아 **+1**. 없으면 `_ver2_`. 즉 `(없음) → ver2 → ver3 → ver4 …` 자동 증가.
  - 예: 디스크에 `회중 성서 연구_훈80-81장_260430.docx`(ver 없음) + `..._ver2_.docx` 둘 다 있으면 다음은 `_ver3_`.

## 4-bis. 일괄 스킬 묶음 확인 절차 (필수)

`midweek-now/next1/next2/next3`, `mid-study1/2/3` 같은 일괄 스킬은 시작 시점에 **반드시** 산출물 현황을 표로 보고하고 한 번 yes/no 받는다. 각 단편 스킬마다 따로 묻지 않는다.

```
{스킬명} ({YYMMDD-MMDD}) 산출물 현황:
| 파트 | 상태 |
|---|---|
| 10분 연설 | ✅ 있음 (ver2) |
| 영적 보물 찾기 | ❌ 없음 |
| 회중 성서 연구 | ✅ 있음 |
| ...

빠진 것만 만들까요? 아니면 있는 것도 다시 만들까요? (있는 것은 _verN_ 새 버전 추가, 기존 보존)
```

**원준님 응답에 따른 분기:**

- 답 없음 / "응" / "빠진 것만" / "그래" → ❌ 만 신규 빌드, ✅ 는 skip
- "다시 만들어" / "전부" / "버전 올려" → ✅ 도 모두 `_verN_` 신규 (N 자동 +1)
- "특정 파트만 다시" (예: "10분 연설만 다시") → 명시 파트만 `_verN_` 신규

**일괄 → 단편 호출 시 mode 전달**: 일괄 스킬은 단편 호출 시 인자 또는 컨텍스트로 결정 사항을 전달 (`--from-batch=skip` 또는 `--from-batch=ver_up`). 단편 스킬은 진입 시 **이 컨텍스트가 있으면 묻지 않고 그대로 실행**한다 (중복 묻기 방지).

자세한 규칙은 `.claude/shared/skip-existing-policy.md` §3 참조 (이 정책의 상위).
