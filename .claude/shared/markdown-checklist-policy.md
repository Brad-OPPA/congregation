# 마크업 체크리스트 정책 (2026-04-25)

> **모든 회중 자료 자동화 에이전트는 작업 진행 시 마크업 체크리스트로 진행 상황을 명시한다.**

## 목적
- 작업 진행 가시화 (어디까지 했는지·무엇이 남았는지)
- 누락 방지 (체크박스 미체크 = 미완료)
- 검수 용이 (다른 에이전트·planner 가 결과 빠르게 파악)

## 형식

각 작업 단계마다 다음 마크업 사용:
- `- [ ]` 미시작
- `- [x]` 완료
- `- [⏳]` 진행 중
- `- [⚠]` 문제 발견 (아래 주석 + 해결 시도)
- `- [❌]` 실패 (아래 사유)

## 산출 위치

각 에이전트는 자기 산출 폴더 (`research-*/<YYMMDD-MMDD>/<part>/`) 에 `_progress.md` 작성:

```
# {에이전트명} — {파트} {YYMMDD-MMDD} 진행

## 단계
- [x] 입력 받기 (브리프·지시서)
- [x] WOL Read (3개 페이지)
- [⏳] 성구 verbatim 검증 (5/8 완료)
- [ ] 출처 URL 기록
- [ ] _selfcheck.md 작성
- [ ] 최종 산출 저장

## 이슈
- [⚠] 성구 X 어디서 인용된 건지 출처 URL 못 찾음 — wol 추가 검색 필요
```

## 정합성

- planner 는 자기 `_planner_review.md` 에서 각 서브 에이전트 `_progress.md` 의 체크 상태 종합
- 미체크 항목 = NEEDS-REWRITE 트리거
- jw-style-checker / fact-checker / timing-auditor 도 자기 검수 단계마다 체크리스트
