## 🚫 할루시네이션 절대 금지 (최상위 규칙)

**이 스킬에서 가장 중요한 규칙. 이 규칙 위반 시 생성물은 사용 불가.**

1. **WOL·신세계역 연구용 성서·JW 출판물에서 실제로 확인한 내용만** 파일에 넣는다. 훈련 데이터 기억이나 추측으로 "「파수대」 2020년 3월호는..." 같은 인용을 만들지 말 것.
2. **확인 못한 내용은 명시적 placeholder** 로 남긴다: `[N항 본문 — WOL 복사]`, `[표어 성구 본문 — 신세계역에서 복사]` 등. 빈 칸이 창작 인용보다 안전하다.
3. **성구 본문은 반드시 신세계역 연구용 성서 verbatim**. 기억으로 쓰지 말 것.
4. **단락 본문은 WOL 기사 페이지 verbatim**. 요약·paraphrase 금지. WebFetch 가 요약만 반환하면 placeholder 로 남길 것.
5. **각 사실 (기사 번호·제목·노래·질문·소제목·복습 질문)** 은 WOL 페이지에서 직접 확인한 다음 적용. WebFetch 는 small-model 래퍼라 환각 가능성 있으니 가능하면 교차 검증.
6. **주중 스킬 (mid-study1/2/3) 의 WOL-first 원칙을 그대로 계승**. 주중에 해결 못 한 것을 주말에서 해결하지 말 것.

**사용자 지정 워크플로 (반드시 따를 것):**

### 절대 규칙

1. **매주 WOL 주차 인덱스부터 시작**: `https://wol.jw.org/ko/wol/dt/r8/lp-ko/YYYY/M/D` (일요일 날짜).
2. **"집회 → 파수대 연구 기사"** 링크를 그대로 따라가서 실제 기사 페이지 (보통 `/d/r8/lp-ko/NNNNNNNNN`) 를 열 것.
3. 질문·성구·단락 본문은 WOL 페이지의 실제 표기 그대로 (ㄱ·ㄴ 구분 포함). 항 번호 합쳐진 경우 ("1-2", "4-5", "7-8") 도 그대로 유지.
4. **관련 출판물 교차 조사는 WOL 검색**: `https://wol.jw.org/ko/wol/s/r8/lp-ko?q=<검색어>` 에서 해당 성구나 주제어로 과거 파수대 기사·통찰 항목 등을 찾아 논지를 메모.
5. 주차 링크 타임아웃 시 3회 재시도. 실패하면 사용자에게 질문.

  - (선택) 새 소제목 — **중앙 정렬 16pt**.
  - **우측 정렬 볼드** 시간 마커 (예: `2'`, `6'`, `9'`, `12'`, ...).
  - 공식 연구 질문 (**이탤릭 14pt**, 일부 노랑 하이라이트) — WOL verbatim.
  - 항 본문 요약 (14pt) — 저작권 고려, verbatim 장문 금지.
  - **깊이있는 해설·교훈점·실생활 예** (노랑 하이라이트 사회자 대사) — ⭐ 심도 리서치 결과가 여기에 반영된다.
  - 성구 낭독 본문 `(책 장:절) 본문` — 주로 **빨강** (신세계역), 일부 **파랑 #0070C0** 강조.
  - (선택) 삽화 설명 / 삽화 캡션.
  - **노랑 하이라이트** "N항입니다." — 다음 항 안내.
- **복습 섹션** — 우측 정렬 시간 마커 → "어떻게 대답하시겠습니까?" (이탤릭 12pt) → 각 복습 질문 (노랑) + 답.
- **결론 블록** — "오늘 기사를 통해," → 교훈 한 문장 (**볼드 노랑**) → 권고 단락.
- **마침** — 감사 말씀 / "오늘의 집회를 모두 마치도록 하겠습니다." / 마침 노래 안내 (노랑) / 기도 담당 (볼드) / 빈 줄 / **우측 정렬 최종 시간 마커** (예: `61"`).

## 산출물 존재 시 skip 정책 (필수)

이 스킬은 자체 3주치 일괄 빌드 — 외부 일괄 스킬(midweek-* / mid-study*) 의 단편 파트로 호출되지 않는다. 따라서 **단독 호출 정책만 적용**.

진입 시 출력 폴더 확인 (3주치 모두):

- **없음** → 정상 진행
- **있음 + 사용자 무명시** → 단정형 확인 1회 ("이미 `파수대 사회_YYMMDD.docx` 가 있는데, 다시 만들까요? 스킵할까요?") — 3주치 산출물 묶어서 한 번에 표로 보고 후 yes/no 1회
  - 답 없음 / "스킵" / "아니" → **skip** (3주치 모두 또는 ❌ 만 빌드)
  - "다시 만들어" / "버전 올려" → `_verN_` 신규 (N = 디스크 최대 + 1, 없으면 ver2)
- **있음 + 사용자가 "재생성·다시·버전 올려" 명시** → `_verN_` 신규 자동 (N 자동 증가)

자세한 규칙: `.claude/shared/skip-existing-policy.md` §3 + `.claude/shared/output-naming-policy.md` §4·§4-bis.
