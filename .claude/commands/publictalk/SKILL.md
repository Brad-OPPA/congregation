# publictalk — 공개 강연 30분 원고 자동 생성 (1편 단위)

## ⭐ 이 스킬의 특징 (다른 스킬과 차이)

- **주중·파수대 스킬과 달리 "3주치" 개념 없음** — 공개 강연은 강연자 로테이션이라 "이번 주 것" 이 고정되지 않는다. 한 번 실행 = **한 강연 번호 1편** 생성.
- **public-talk-builder 가 orchestrator 역할** — 단순 아웃라인 제공자가 아니라 **"기획자 (architect)"** 로서 30분 원고 설계도를 만들고, 그 설계도 안에 **"각 요점에 다른 에이전트가 수집할 재료 명세서"** 까지 작성. 다른 에이전트들은 그 명세서대로 타겟된 재료만 수집 → 원고 품질이 높아짐.
- **골자 소스는 로컬 PDF** — `docs.jw.org` 는 로그인 필요해서 자동 fetch 불가. 사용자가 이미 다운로드해 둔 PDF 를 Read.

## 🎙️ 대화형 시작 흐름 (사용자 지정, 필수 준수)

사용자가 `/publictalk` 또는 "공개강연 만들어 줘" 를 치면:

### 1단계 — 강연 번호 묻기
> "몇 번 공개 강연 원고를 만들어 드릴까요? (1~194 중 번호)"

### 2단계 — 로컬 골자 확인 및 골자 소스 확인
번호를 받으면 즉시 **두 형식 모두** 검색:
```
# 신형 골자 (S-34 시리즈, .docx)
C:\Users\yoone\Dropbox\02.WatchTower\02.▣ 집회(Meetings)\S01.공개강연\골자\S-34_KO_{번호:03d}.docx
# 구형 골자 (PB 시리즈, .pdf)
C:\Users\yoone\Dropbox\02.WatchTower\02.▣ 집회(Meetings)\S01.공개강연\골자\PB_{번호:03d}-KO*.pdf
```
**골자 폴더 직속에 둠 — `KO/` 하위 폴더 없음.** 두 형식 중 어느 것이든 발견되면 Case A.

**Case A — 로컬 골자 있음**:
- 신형 (`S-34_KO_NNN.docx`) 우선 사용 (최신 개정판이 이 형식으로 배포됨)
- 둘 다 있으면 신형 우선, 사용자에게 "신형 S-34 골자를 사용합니다 (구형 PB_NNN.pdf 도 있지만 최신본 우선)" 안내
> "로컬에 `S-34_KO_{NNN}.docx` (또는 `PB_{NNN}-KO*.pdf`) 골자 파일이 있습니다. 이 골자로 작성할까요? 아니면 최신 골자를 직접 주시겠어요?"

사용자 선택:
- (a) "있는 골자로" / "로컬" / "이걸로" → 로컬 파일 Read → 3단계로
- (b) "새 골자 주겠다" / "최신으로" → 사용자가 골자 텍스트/파일 경로 제공할 때까지 대기 → 받으면 3단계로

**Case B — 로컬 골자 없음**:
> "로컬에 `S-34_KO_{NNN}.docx` 또는 `PB_{NNN}-KO*.pdf` 골자 파일이 없습니다. `docs.jw.org/ko/-/cds-cat-docs-outlines` 에서 해당 번호 골자를 다운로드해서 `S01.공개강연/골자/` 에 넣어주시거나, 골자 텍스트를 직접 붙여넣어 주세요."

사용자가 파일 넣거나 텍스트 제공할 때까지 대기.

### 3단계 — 골자 Read 및 제목 추출
- **신형 .docx**: `python -X utf8 -c "from docx import Document; ..."` 로 paragraphs 추출 (cp949 인코딩 회피 위해 `-X utf8` + `sys.stdout = io.TextIOWrapper(..., encoding='utf-8')` 필수)
- **구형 .pdf**: Read 도구 (Claude 는 PDF 직접 읽기 가능)
- 사용자 제공 텍스트라면 그대로 사용
- 제목·표어 성구·요점 추출 → 사용자에게 1줄 확인:
  > "번호 NNN · 제목 '{제목}' · 표어 성구 {성구} 로 진행하겠습니다. 리서치 시작합니다..."

### 4단계 이후 — 아래 [실행 파이프라인] 진행 (자동)

**예외**: 사용자가 처음부터 `/publictalk 179` 처럼 번호를 같이 주거나, `/publictalk 179 로컬` 처럼 소스까지 주면 1-2단계 스킵 가능. 단 제목·표어 성구 확인(3단계 마지막 줄)은 **반드시** 거쳐서 빨리 잘못 가는 걸 막을 것.

## 🚫 할루시네이션 절대 금지 (최상위 규칙)

이 스킬이 호출하는 모든 Agent (public-talk-builder · scripture-deep · publication-cross-ref · illustration-finder · experience-collector · application-builder · jw-style-checker · timing-auditor) 는 아래를 준수:

1. **공식 JW 출판물·신세계역 연구용 성서·골자 PDF 에서 실제로 확인한 내용만** 산출물에 넣는다. 훈련 데이터 기억·추측으로 인용·경험담·예화를 만들지 말 것.
2. **확인 못 한 항목은 명시적 placeholder** `[확인 필요]`. 빈 칸이 창작 인용보다 안전하다.
3. **모든 인용은 출처 URL + 출판물 이름/호수/면/항** 필수. URL 없는 인용 금지.
4. **성구 본문은 신세계역 연구용 성서 verbatim.**
5. **경험담은 공식 출판물에 실제 게재된 것만.** 창작 금지.
6. **예화·비유도 출처 명시 필수.**

**모든 Agent 호출 프롬프트 말미에 다음 공통 문구를 반드시 첨부:**

> ⚠ 할루시네이션 절대 금지: 훈련 데이터 기억이 아니라 공식 JW 출판물 (wol.jw.org) · 신세계역 연구용 성서 · 사용자가 제공한 골자 PDF 에서 **실제로 확인한 내용만** 저장할 것. 확인 못 한 항목은 `[확인 필요]` placeholder. 모든 인용은 출처 URL + 출판물 이름/호수/면/항 포함. 성구는 verbatim. 경험담·예화도 출처 명시 필수.

jw.org 및 wol.jw.org 는 공개 자료이므로 저작권은 고려하지 않는다 (사용자 지침, 2026-04-22). 파수대·깨어라·통찰·훈 책 등의 장문 verbatim 인용·단락 전문 복사 모두 허용. 단, 출처 URL 명시는 할루시네이션 방지 차원에서 여전히 필수.

JW 출판물(파·깨·통찰)에 실린 예시도 사용 가능. 단 그대로 인용 금지 — **외부 1차 자료로 한 걸음 더 구체화**해 출판물 문장 대비 구체성 2배 이상.
- 예: 출판물 "키루스 실린더가 이사야 예언과 일치" → 확장 "1879년 대영박물관 라삼이 바빌론에서 발견한 22.5cm 토기 원통, 쐐기문자로…"
- 방법: 출판물이 인용한 원출처를 역추적(발굴 연도·박물관·발굴자·치수·원문)

### ✨ 목표 효과 — 청중 묵상·심화 촉발 체크리스트
서론 소재는 단순 흥미가 아니라 **"어, 이런 건 생각 못 했다"** 깨달음을 주고 **묵상·성서 재독 욕구** 를 촉발해야. 4개 체크 중 ≥3개 "예" 여야 통과:
- [ ] 청중이 **전에 알지 못했던** 사실·장면인가?
- [ ] 성서·여호와의 지혜와 자연스럽게 이어지는가?
- [ ] 듣고 나서 "더 찾아보고 싶다"/"해당 성구 다시 읽고 싶다" 마음이 생기는가?
- [ ] 본 강연 전체를 들은 후에도 서론의 깨달음이 머릿속에 남는가?

### 🔍 외부 자료 적절성 검수 필수 — jw-style-checker + fact-checker
외부 자료는 자유 사용 허용하지만 **JW 교리·원칙·정중성과 부합**해야. 아래는 자동 탈락:
- 진화론을 사실로 전제 / 정치 편향 / 타 종교 교리 긍정 / 성적·폭력적 장면 / 출처 불명 전승 / 음모론 / 낙태·동성애·백신 등 논쟁적 현대 사안 / 비증인 설교자의 견해 권위 인용

설계도에 **"서론 소재 적절성 체크"** 표 의무 기록 → fact-checker 가 1차 자료 교차 검증, jw-style-checker 가 교리·톤 감수.

### 🏋️ 서론·예 리서치 물량 하한 (원준님 지침 2026-04-24)
**"서론이나 예가 엄청 중요함"**. 후보를 많이 뽑아 놓고 최상만 고르는 방식으로만 품질이 나옴.
- **서론 후크 후보**: 최소 **15~20개** 발굴 → TOP 3 선정 → 3안 제시
- **14개 축 중 최소 10개 축** 의무 탐색 (9개 성서 적중 + 5개 사유 촉발)
- **요점당 예화 후보**: 5~7개 → TOP 2
- **요점당 경험담 후보**: 3~5개 → TOP 1
- 하한 미달이면 설계도 미완성 → Planner 재검수에서 보강

### 🤔 "생각해 볼만한" 사유 촉발형 후크 5개 축
성서 적중 입증형 9개 축 외에 아래 5개 축도 허용 (영적 성품·관계·감정 주제에 적합):
- 🤔 **인생 의미·가치관 질문** — 후회·선택·목표
- ⏳ **시간·유한성 자각** — 남은 일수·연령 반추
- 🪞 **자기 점검형** — 일상 습관·내면 질문
- 🧭 **선택의 무게** — 한 번의 결정이 바꾼 삶
- 💔 **상실·회복 공감** — 누구나의 감정 → 성서의 위로

단, 반드시 마지막에 성서·여호와의 지혜로 귀결. 애매한 자기계발 강연처럼 끝나지 말 것.

### 서론 설계 체크리스트
- [ ] 첫 문장에 "안녕하십니까/시작하기 전에/오늘 주제는" 없음
- [ ] 첫 3문장 안에 **구체적 숫자·연도·장면·발언** 중 최소 1개

- [ ] 후크 → 문제 제기 → 표어 성구 낭독 → 강연 예고 순서
- [ ] 인사말 분량 0초 (처음부터 본론 몰입)

## 🎨 원고 시인성 강조 마크업 (빌더 지원 문법)

`build_publictalk.py` 는 아래 인라인 마크업을 지원. 원고 작성 시 **적극 사용**해 강연자가 손대지 않고 낭독할 수 있도록 강조 포인트를 미리 표시.

| 마크업 | 효과 | 사용 지점 |
|---|---|---|
| `==text==` | **볼드 + 노랑 하이라이트** | 수사적 질문, 핵심 문장, 감정 몰입 장면, 시각 자료 지시 |
| `!!text!!` | **빨강 볼드** | 연도, 인명, 핵심 개념, 강조 단어 |
| `**text**` | **볼드** | 일반 강조, 요점 단어 |
| `*text*` | *이탤릭* | 성구 낭독 본문 (인용 블록 `>` 와 함께) |
| `` `[N'NN"]` `` | 빨강볼드 + 노랑 (단독 라인 우측 정렬) | 시간 마커 |
| `> *성구*` | 이탤릭 회색 인용 블록 | 성구 verbatim 낭독 |

### 사용 비율 가이드
- `==노랑 하이라이트==` 는 **단락당 0~2개** (남발하면 시인성 저하)
- `!!빨강 볼드!!` 는 단락당 1~3개 가능 (연도·인명·핵심 개념에 밀집 가능)
- 모든 수사적 질문에는 `==...==` 또는 `**...**` 강조 필수
- 시각 자료 지시문은 반드시 `==[시각 자료 N — 설명]==` 형식

## ⭐ 원고 문체 기준 — "2024 스타일" (필수 준수)

**레퍼런스**: `2024 특별강연_김원준20240314_Final.pdf` ("부활_죽음에 대한 승리") — 사용자의 최신·완성형 문체. 이 기준에 미달하면 원고 불합격.

### 1. 문장 형태
- ✅ **완전한 서술형 구어체** — 모든 문장이 사회자/강연자가 그대로 읽을 수 있는 자연스러운 문장
- ❌ 메모형·개요형·대시 축약 금지 (예 `"욕망이 자라면 죄를 낳고 _"` 같은 표기 금지)
- ✅ 문장 끝은 `~합니다 / ~하겠습니다 / ~입니까? / ~이 아닙니까?` 같은 강연체

### 2. 청중 인터랙션 (필수, 매 2~3분마다 1개)
- "혹시 오늘 처음 오신 분이 계신다면 옆에 계신 분께 요청하셔서 함께 보실 수 있겠습니다."
- "주제가 기억나십니까?"
- "잠시 눈을 감아 보실까요?"
- "∼라고 생각하시나요?" / "∼이 아닙니까?"
- 성구 낭독 전 유도: "같이 신세계역 성서, {책 장:절} 보시겠습니다."

### 3. 시간 마커 — 30초 단위 정밀도
- 분·초 단위 (예: `4"`, `5'30"`, `7'30"`, `10'`, `12'`, `13'30"`, `16'`, `18'30"`, `21'30"`, `25'`, `27'30"`, `29'30"`)
- **우측 정렬, 빨간 볼드, 노란 하이라이트**
- 요점 전환 시·시각자료 표시 시·동영상 시작 시 각각 시간 마커

### 4. 일상 비유 최소 3개 (요점당 1개)
친숙한 것 → 우주적인 것으로 연결. 레퍼런스의 예:
- 진시황 불로초 (영생 열망)
- 친구가 50억 준다는 말 (믿기 어려움)
- 도로변 지하 15층 건축 (희생의 규모)
- 먼 곳 친구 초대 비용 (그리움·능력)

### 5. 감정 몰입 장면 (결론부 필수 1개)
- "잠시 눈을 감아 보실까요?" 로 시작
- 청중이 머릿속에 상상할 수 있는 구체적 장면 (시간·거리·인물·감정)
- 예: "이제 천년 통치가 시작되고 120년 정도 지났습니다. 멀리서 어렴풋이 사람의 모습이 보입니다…"

### 6. 수사적 질문 밀도
- 매 문단에 1개 이상 권장
- **노랑 하이라이트**로 표시
- 답을 청중에게 맡기거나 직접 이어서 답해줌

### 7. 하이라이트 3색 체계 (docx 포맷)
- 🟡 **노랑 하이라이트** = 수사적 질문, 핵심 문장, 사회자 대사 (강조할 구어체)
- 🔴 **빨강** = 강조 단어 (`EE0000`), 성구 내 핵심 키워드
- _이탤릭_ = 성구 낭독 본문 (`회색 박스` 또는 들여쓰기)
- **볼드** = 성구 참조 (예: `고린도 전서 15:26`), 중요 개념

### 8. 전환구의 자연스러움
요점·성구·예화 간 전환 시:
- "그렇다면…" / "그런데…" / "혹시…" / "그에 더해…" / "마지막으로…"
- "같이 … 보시겠습니다." / "이 성구에서 … 묘사하고 있습니까?" / "어떤 의미입니까?"

### 9. 성구 활용 (30분 표준)
- **낭독 성구 6~8개** (레퍼런스: 8개)
- **언급 성구 4~6개** (언급만 하고 넘어감)
- 각 성구 전에 **질문 1개 + 유도 1문장** → 낭독 → **해설 2~3문단**
- 성구 유도문: "그 점을 … 에서 확인해 보겠습니다" / "같이 … 보시겠습니다"

### 10. 시각자료 / 동영상 — 소스 우선순위 + 총 상한 (사용자 지침 2026-04-24)

#### 🔒 10-0. 최우선 공통 규칙 (사용자 지침 2026-04-24 확정판)

> **"종교적 내용이 있는 이미지는 오직 wol.jw.org 에서만. 외부 이미지는 세속적·중립적 사실(고고학 유물·지형·자연·과학·일상 사물)만."**

종교성 이진 판정: 성경 장면·영적 상징·성인·예배 의식·내세관 중 하나라도 해당 → **외부 불가, wol 전용**. wol 에도 없으면 삽화 생략. 타 종교 전통 회화(서양 종교화·동방 이콘·가톨릭 성화·불교 불화·힌두 신상·몰몬 공식 일러스트 등) 전부 자동 탈락. 179 V3 의 Danby「대홍수」·Bruegel 풍 바벨탑 류는 불가.


#### 10-1. 총 상한 — **최대 5장**
- 30분 공개 강연 1편당 **임베드 이미지 최대 5장** (`![](...)` 로 원고에 넣는 것 기준)
- 권장 배분: **서론 0~1 + 요점 1·2·3 각 1 + 결론 0~1**
- JW 방송 동영상 1건은 이미지와 별개로 계산 (최대 1건 권장)
- 6장 이상은 timing-auditor 가 경고 — 메시지 분산 위험
- 골자 PDF 에 명시된 공식 "필수" 삽화는 반드시 포함, 나머지 슬롯을 아래 규칙대로 채움

#### 10-2. 소스 우선순위 — 4단계 (순서 준수 의무)
```
1순위: wol.jw.org 기사 본문 삽화
       · 해당 강연 골자 PDF 에 명시된 공식 삽화
       · 관련 파수대/깨어라/훈/가까이하라/하느님의 사랑 기사의 본문 삽화
2순위: jw.org 도서관·기사 이미지 (jw.org/ko 내 공식 일러스트)
3순위: JW 방송 영상 캡처 장면 (tv.jw.org / jw.org/ko 동영상)
4순위: 외부 이미지 — 위키커먼즈 · 박물관 공개 고해상도 · 공개 도메인
       ※ 상업 스톡(Shutterstock·Getty·iStock 등) 금지
       ※ 4순위 사용 시 "외부 대체 후보" 플래그 + 사용자 승인 포인트 명시
```

**1~3순위를 실제로 검색한 기록(URL)이 설계도에 남아 있어야** 4순위를 채택할 수 있다. 건너뛰고 바로 외부로 가는 것은 규칙 위반.

#### 10-3. 유형별 엄격도 — A / B 구분

| 유형 | 예시 | wol 우선도 |
|---|---|---|
| **A. 후크·예화 삽화** (사물·역사·자연·일상 비유) | 체임벌린 신문, 도쿄 네온사인, 마라톤 주자, 진시황, 스마트폰 | **권장** — 가능하면 wol/JW방송 먼저 조회. 비슷한 게 없으면 외부(4순위) 허용 |

- **B 유형**: 1~3순위에서 찾을 때까지 검색 확대 (기사 본문 스크롤·관련 파수대 연도 확장·JW방송 드라마 캡처). 그래도 없을 때만 4순위, 각 건마다 사용자 승인 필요
