# mid-study3 — 회중성서연구 3주치 일괄 (오케스트레이터)

> **출력 경로·파일명 정본**: `.claude/shared/output-naming-policy.md` (2026-04-25 통일).

## 이 스킬의 범위
- **이번 주 + 다음 주 + 다다음 주** 3주 분량 회중성서연구 사회 (30분) 일괄 생성
- 각 주차당 1편 → 총 3편 docx + PDF 산출
- 모든 빌드 로직(WOL 리서치·할루시네이션 방지·감수 게이트·docx 렌더)은 단편 스킬 **`/cbs`** 본문에 있음. 이 스킬은 **얇은 오케스트레이터**.

## 산출물
- `/cbs` 의 출력 그대로 (3주차분):
  - `회중 성서 연구_훈{장}-{장}_YYMMDD.docx` + `.pdf`
  - 저장 경로: `…\01.주중집회\05.회중 성서 연구\YYMMDD_M월 D-D일\`

## 실행 단계

### 0. 묶음 확인 (산출물 존재 시 skip — 시작 시점 필수)

3주차 산출물 현황을 표로 한 번에 보고하고 yes/no 받음. **각 주차마다 따로 묻지 않는다**.

```
mid-study3 (회중 성서 연구) 3주치 산출물 현황:
| 주차 | YYMMDD | 상태 |
|---|---|---|
| 이번 주 | YYMMDD | ✅ 있음 (ver2) / ❌ 없음 |
| 다음 주 | YYMMDD | ... |
| 다다음 주 | YYMMDD | ... |

빠진 것만 만들까요? 아니면 있는 것도 다시 만들까요? (있는 것은 _verN_ 새 버전 추가, 기존 보존)
```

- 답 없음 / "응" / "빠진 것만" / "그래" → ❌ 만 신규 빌드, ✅ 는 skip
- "다시 만들어" / "전부" / "버전 올려" → ✅ 도 모두 `_verN_` 신규 (N = 디스크 최대 + 1, 없으면 ver2)
- "특정 주차만 다시" → 명시 주차만 `_verN_` 신규

**자리 비움 시 안전:** 답 없으면 자동으로 skip — 검수·발송된 docx 가 의도치 않게 덮어쓰이지 않는다.

### 1. 단편 스킬 순차 호출 (묶음 컨텍스트 전달)

§0 결정에 따라 호출:

- **skip 결정 주차** → `/cbs` 호출 자체 발생 안 함
- **신규 빌드 주차** → `Skill(cbs, args="<주차>")`
- **ver_up 주차** → `Skill(cbs, args="<주차> --from-batch=ver_up")`

호출 예 (3주 모두 신규):
1. `Skill(cbs, args="now")`
2. `Skill(cbs, args="next1")`
3. `Skill(cbs, args="next2")`

`/cbs` 는 묶음 컨텍스트 (`--from-batch=…`) 를 받으면 **자체 단정형 확인 묻지 않고** 그대로 실행. 각 호출 결과 경로를 누적.

### 2. 종합 보고
```
✅ 회중성서연구 3주치 생성 완료
- 이번 주:    회중 성서 연구_훈{장}-{장}_YYMMDD.docx (+ .pdf)
- 다음 주:    회중 성서 연구_훈{장}-{장}_YYMMDD.docx (+ .pdf)
- 다다음 주:  회중 성서 연구_훈{장}-{장}_YYMMDD.docx (+ .pdf)
```

### 3. 에러 처리
- 특정 주차 실패 → 해당 주차만 재시도 (3회), 나머지 주차는 계속
- 3회 후에도 실패 → 원준님께 에러 보고, 나머지 주차는 정상 완료

## 산출물 존재 시 skip 정책 (필수)

스킬 진입 시 출력 폴더에 산출물이 이미 있는지 먼저 확인한다. 일괄(3주치) 스킬이므로 §0 묶음 확인 절차로 한 번에 처리한다.

- **없음**: 정상 진행
- **있음 + 사용자 무명시**: §0 묶음 확인 표 1회 → 답 없거나 No → **skip**
- **있음 + 사용자가 "재생성·업그레이드·버전 올려·다시 만들어" 명시**: 파일명 끝에 `_verN_` 부여 후 신규 생성 (기존 파일 보존). **N 자동 결정** — 같은 폴더에서 같은 prefix 의 `_verN_` 중 최댓값 +1, 없으면 `_ver2_`.

`/cbs` 단편 스킬은 이 일괄 스킬에서 호출될 때 `--from-batch=…` 컨텍스트를 받아 **자체 단정형 확인 묻지 않고** 그대로 실행 — 중복 묻기 방지.

자세한 규칙: `.claude/shared/skip-existing-policy.md` §3 + `.claude/shared/output-naming-policy.md` §4·§4-bis.
